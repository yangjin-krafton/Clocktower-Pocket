# Clocktower Pocket

오프라인 소셜 디덕션 보드게임 **"시계탑에 흐른 피(Blood on the Clocktower)"** 를 사내에서 진행할 때, 종이/토큰/그리모어 같은 **물리 소품을 휴대폰 웹 UI로 대체**하기 위한 **현장 보조용 웹 앱 샘플**입니다.
게임의 핵심(현장 토론, 처형, 밤/낮 흐름, 스토리텔러 진행)은 그대로 두고, **정보 전달/투표/타이머/기록**만 디지털로 보조합니다. 
---

## 목표(Goals)

* 참가자는 **개인 휴대폰 브라우저**로 접속해 자신의 상태(생존/사망), 알림, 투표, (허용되는 범위의) 개인 정보 등을 확인
* 스토리텔러는 **관리자 화면(그리모어 대체)** 에서 배역/밤 액션/사망/처형/투표 결과를 관리
* **서버 없이** GitHub Pages에서 정적으로 배포 — P2P(WebRTC)로 플레이어 간 직접 통신

## 비목표(Non-goals)

* 온라인(원격) 완전 자동 진행 게임을 목표로 하지 않음(토론/심리전은 오프라인 현장 중심)
* "정식 룰북/정식 에셋" 제공 및 배포 목적이 아님
* 별도 서버 운영(Node.js, DB 등) — 모든 상태는 스토리텔러 브라우저에서 관리

---

## 게임 진행 개요(현장 흐름)

본 샘플은 기본적으로 아래 흐름을 지원합니다.

* **밤:** 스토리텔러가 순서대로 역할 능력을 처리(플레이어는 휴대폰으로 "호출/선택/확인"만 수행 가능)
* **낮:** 현장 토론 → 지명/투표 → 처형 처리(웹에서 투표 수집/집계/로그)
* 일반적으로 **선 팀은 데몬 처형**, **악 팀은 종료 조건(예: 생존 3인)** 달성에 목표

> 입문 스크립트는 Trouble Brewing을 기본 타겟으로 가정(필요 시 스크립트 교체)

## 시스템 구성(Architecture)

### 전체 구조

```
[스토리텔러 브라우저]  ←── WebRTC(Trystero) ──→  [플레이어 브라우저들]
  · 게임 상태 보유          BitTorrent 트래커          · 개인 화면
  · 그리모어 UI             (시그널링 중계)             · 역할/투표/밤행동
  · 방 생성/코드 발급
```

* **시그널링**: [Trystero](https://github.com/dmotz/trystero) — BitTorrent 공개 트래커를 시그널링 서버로 활용 (별도 서버 불필요)
* **연결**: WebRTC 직접 P2P — 이후 트래커와 연결 끊어도 게임 지속
* **게임 상태**: 스토리텔러(Host) 브라우저 메모리에 상주, `sessionStorage`에 주기적 백업
* **배포**: GitHub Pages (정적 HTML/CSS/JS)

### 메시지 타입

| 방향 | 타입 | 내용 |
|---|---|---|
| Host → 전체 | `PHASE_CHANGE` | 밤/낮 전환, 투표 시작/종료 |
| Host → 전체 | `PLAYER_DIED` | 사망/처형 알림 |
| Host → 전체 | `VOTE_RESULT` | 투표 집계 결과 |
| Host → 특정 플레이어 | `ROLE_ASSIGN` | 역할 배정 (비공개) |
| Host → 특정 플레이어 | `NIGHT_RESULT` | 밤 행동 결과 (비공개) |
| Host → 특정 플레이어 | `NIGHT_CALL` | 밤 행동 요청 |
| 플레이어 → Host | `NIGHT_ACTION` | 밤 행동 제출 |
| 플레이어 → Host | `VOTE` | 찬/반 투표 |
| 플레이어 → Host | `JOIN_REQUEST` | 닉네임 포함 입장 요청 |

### 1) 클라이언트(참가자)

* 모바일 우선(Responsive) 웹 UI
* 기능

  * "나의 상태": 생존/사망, 남은 투표권(사망 후 유령 투표 1회 등 룰 옵션)
  * "알림": 밤 행동 요청/확인, 낮 투표 시작/종료, 처형 결과
  * "투표": 찬/반 또는 지명자 선택(진행 방식에 따라)
  * "비공개 정보": 본인 역할(또는 스토리텔러 설정에 따른 비공개 정보 표시)

### 2) 관리자(스토리텔러)

* 데스크톱/태블릿 최적
* 기능

  * 방 생성/코드 발급(6자리), QR 코드 표시
  * 플레이어 등록/좌석(원형 시계 방향) 관리
  * 배역 배정(랜덤/수동), "블러프(악팀 가짜 역할)" 메모
  * 밤 페이즈 진행(액션 큐/프롬프트)
  * 사망/처형 처리, 투표 집계/결과 확정
  * 로그(밤 행동/정보 제공 내역), 되돌리기(Undo) 옵션

### 3) 서버 — 없음

| 구분 | 기존 구조 | 현재 구조 |
|---|---|---|
| 호스팅 | Node.js 로컬 서버 | GitHub Pages (무료 정적) |
| 실시간 통신 | WebSocket / Socket.IO | WebRTC (Trystero) |
| 시그널링 | 자체 서버 | BitTorrent 공개 트래커 |
| 게임 상태 | 서버 메모리 / DB | 스토리텔러 브라우저 |
| 접근 | `http://로컬IP:8080` | `https://yourname.github.io/...` |

## "소품 대체" 매핑

| 오프라인 소품        | 모바일/P2P 대체                          |
| -------------- | ------------------------------------- |
| 캐릭터 토큰/시트      | 개인 화면 "역할 카드"(텍스트는 요약/링크)           |
| 그리모어(스토리텔러 보드) | 관리자 "그리모어 UI"(좌석/토큰/상태/메모)          |
| 밤에 깨우기/선택      | 개인 화면 알림 + 선택 UI (Host가 P2P로 전송)    |
| 투표 손들기         | 모바일 투표 버튼 + 타이머 + 집계 (Host가 집계)     |
| 사망 표시 토큰       | 상태 토글(생존/사망) + P2P 전체 공지            |
| 타이머/벨          | Host 브라우저 타이머 브로드캐스트                |

* [ ] 플레이어 닉네임/좌석 확정

## 화면 정의(초안)

GitHub Pages는 SPA가 아니므로 URL 파라미터(`?code=ABC123`)로 방 코드 전달.

### 참가자

* `index.html?join` : 방 코드 입력 / QR 스캔
* `player.html?code=ABC123` : 내 역할 / 상태 / 알림 / 투표 통합 화면

### 스토리텔러

* `host.html?code=ABC123` : 룸 대시보드 + 그리모어 + 밤/낮 진행 + 투표 제어

## 룰/운영 옵션(스토리텔러 설정값)

현장마다 운영 방식이 달라질 수 있어 옵션화합니다.

* 투표 방식: "지명자 1명에 대한 찬/반" vs "후보 다수 중 선택"
* 사망자 참여: 발언 가능/불가, 유령 투표 사용 여부(기본 룰은 설정에 따라 다를 수 있어 토글)
* 정보 노출: 역할 텍스트를 전부 노출 vs 요약 + 참조 링크
* 로그 공개: 스토리텔러만 vs 종료 후 리플레이 공개

## 네트워크 체크리스트

* 모든 기기가 **인터넷에 연결**되어 있어야 함 (BitTorrent 트래커 접근용 — 이후 게임 중엔 불필요)
* 회사 네트워크에서 BitTorrent 포트 차단 시 → Trystero `nostr` 전략으로 교체 가능
* 스토리텔러 탭을 닫으면 게임 상태 소멸 → sessionStorage 백업 권장
* 동시 접속 권장 인원: **5~15명** (Trystero 안정 범위, BotC 적정 인원과 일치)

## 보안/프라이버시

* WebRTC 채널은 DTLS로 암호화됨 — 제3자 도청 불가
* 역할/비공개 정보는 Trystero의 `peerId` 지정 전송으로 **해당 플레이어에게만** 전달
* 룸 코드는 충분히 랜덤(6자리 영숫자)하게 생성, 행사 종료 시 자연 소멸
* 로그는 기본적으로 스토리텔러만 열람
