# Clocktower Pocket

Clocktower Pocket은 **Blood on the Clocktower 현장 진행을 돕는 정적 브라우저 프로토타입**입니다.  
현재 구현은 "이야기꾼이 한 화면에서 게임을 진행하고, 참가자는 각자 휴대폰에서 자기 역할과 규칙을 확인하는 보조 도구"에 가깝습니다.

초기 문서에서는 WebRTC/P2P 기반 실시간 동기화를 크게 다뤘지만, 지금 코드베이스의 중심은 그 경로가 아닙니다. 실제 앱은 **자리별 역할을 인코딩한 방 코드**를 생성하고, 참가자가 **방 코드 + 자리 번호**로 자기 역할을 확인하는 흐름으로 정리되어 있습니다.

## 지금 이 프로젝트는 무엇인가

- `src/`만으로 배포되는 정적 HTML/CSS/JS 앱입니다.
- 호스트 화면은 Trouble Brewing 게임 준비, 밤/낮 진행, 판정 로그, 히스토리 되돌리기를 담당합니다.
- 참가자 화면은 자기 역할 카드, 규칙 문서, 메모, 자리 추리 표시를 담당합니다.
- 서버나 데이터베이스는 없고, 현재 게임 상태와 세이브는 브라우저 `localStorage`에 저장됩니다.

## 이 프로토타입이 실제로 구현한 것

### 호스트 앱

- 5명부터 15명까지 인원 설정
- Trouble Brewing 역할 수동 배정 / 자동 배정
- 자리별 역할과 Fortune Teller 레드 헤링을 포함한 방 코드 생성
- 첫밤 / 반복밤 순서 진행
- 정보 전달형 역할과 선택형 역할 처리
- Minion 공개, Demon 블러프 3개 제안 및 전달
- 낮 지목, 투표 수 수동 입력, 처형 판정
- Slayer, Virgin, Saint, Monk, Imp, Scarlet Woman 등 주요 상호작용 처리
- 로그 기록, 히스토리 바, 특정 시점 되돌리기
- 게임 세이브 / 복원

### 참가자 앱

- 방 코드와 자리 번호로 입장
- 자기 역할 카드와 역할별 규칙 문서 확인
- Trouble Brewing 캐릭터 사전 열람
- 개인 메모 저장
- 자리 배치 화면에서 의심 마크 표시
- 마지막 참가 세션 / 저장된 게임 이어하기

## 이 프로토타입이 실제로 이렇게 바뀌었다

| 초기 기획 / 오래된 문서 | 현재 구현 | 독자가 알아야 할 점 |
|---|---|---|
| WebRTC + Trystero 기반 실시간 호스트-참가자 동기화 | 방 코드 인코딩 / 디코딩 중심 오프라인 흐름 | 참가자 앱은 실시간 게임 클라이언트가 아니라 개인 참고 도구에 가깝습니다. |
| 참가자가 입장하고 상태를 계속 수신 | 참가자가 자기 역할만 코드로 복원 | 생존/사망/투표 같은 상태는 자동 동기화되지 않습니다. |
| P2P 연결이 핵심 인프라 | 호스트 브라우저가 정본 | 현장 진행은 호스트 화면 중심으로 돌아갑니다. |
| 확장 가능한 멀티 스크립트 / 넓은 자산 계획 | 실제 엔진과 규칙은 Trouble Brewing 중심 | 아이콘 자산이 더 많아 보여도 게임 로직의 기준은 TB입니다. |

## AI가 쓰인 지점

현재 배포 앱에는 **LLM 호출이나 외부 AI API 연동이 없습니다**.

대신 호스트 진행을 돕는 로컬 조언 모듈이 들어 있습니다.

- `src/js/host/NightAdvisor.js`
  밤 정보 역할에 대해 "정확한 정보 / 애매한 정보 / 악팀 도움 / 직접 선택" 같은 선택지를 제안합니다.
- `src/js/host/DemonBluffAdvisor.js`
  현재 필드 상태를 보고 임프 블러프 3개 세트를 추천합니다.

즉, 이 저장소에서의 "AI 느낌"은 모델 호출이 아니라 **호스트 판단을 보조하는 규칙 기반 추천 로직**에 더 가깝습니다.  
`docs/devlog/`에는 LLM과 함께 작업한 흔적이 남아 있지만, 그 내용이 곧 런타임 기능을 뜻하지는 않습니다.

## 무엇이 프로토타입으로서 잘 됐는가

- 별도 빌드 없이 GitHub Pages에 올릴 수 있는 구조로 단순화됐습니다.
- 호스트가 실제 테이블 진행을 따라가며 밤/낮을 운영하기 좋게 화면이 정리됐습니다.
- 규칙 문서를 Markdown으로 분리해 역할 카드와 사전을 같은 앱 안에서 열 수 있습니다.
- 저장, 히스토리, 되돌리기까지 포함돼 실전 진행 실수 복구에 유리합니다.

## 무엇이 아직 미완성인가

- 실시간 참가자 상태 동기화는 현재 게임 루프에 연결되어 있지 않습니다.
- 닉네임 기반 로비 입장보다 자리 번호 기반 흐름이 우선이라, 현장 운영 편의는 있지만 온라인성은 약합니다.
- 투표는 자동 수집이 아니라 호스트 수동 입력입니다.
- 참가자용 `Join`, `Waiting`, `EmojiPanel`, `PlayerTracker`, `p2p.js` 같은 파일은 남아 있지만 현재 메인 진입 흐름에서 적극 사용되지 않습니다.
- 여러 공식 스크립트를 플레이하는 범용 엔진이 아니라 Trouble Brewing 프로토타입에 가깝습니다.

## 실행 방법

이 저장소에는 `package.json`이 없고, 별도 빌드 단계도 없습니다.  
GitHub Pages 배포도 `.github/workflows/deploy.yml`에서 **`src/` 폴더를 그대로 업로드**하는 방식입니다.

로컬에서 보려면 정적 서버 하나면 충분합니다.

```bash
python3 -m http.server 4173 --directory src
```

브라우저에서 `http://127.0.0.1:4173`를 열면 됩니다.

`file://`로 바로 열어도 일부 화면은 보일 수 있지만, 규칙 문서는 `fetch()`로 읽기 때문에 정적 서버 실행이 더 안전합니다.

## 폴더 가이드

- `src/index.html`: 단일 진입점
- `src/js/app.js`: 홈 화면과 저장된 게임 목록
- `src/js/host/`: 호스트 UI와 진행 흐름
- `src/js/player/`: 참가자 UI
- `src/js/game-engine.js`: Trouble Brewing 규칙 엔진
- `src/js/data/roles-tb.js`: 역할 데이터와 밤 순서
- `src/js/room-code.js`: 방 코드 인코딩 / 디코딩
- `src/js/GameSaveManager.js`: 저장 슬롯 관리
- `src/rules/`: 역할 규칙 Markdown
- `docs/`: 설계 문서와 실험 기록
- `src/js/p2p.js`: 과거 P2P 실험 코드

## 법적 주의와 프로젝트 성격

- **Blood on the Clocktower**는 The Pandemonium Institute의 저작물입니다.
- 이 저장소는 공식 앱이 아니라 **오프라인 현장 진행을 보조하는 비공식 프로토타입**입니다.
- 저장소에는 공식 아트워크 전체를 포함하지 않는 방향을 전제로 하고 있습니다.
- 실제 플레이와 배포는 원작 게임의 권리와 라이선스를 따로 확인해야 합니다.
