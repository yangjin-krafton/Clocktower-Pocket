# Blood on the Clocktower 통합 규칙서 (한국어)

이 문서는 아래 자료를 통합한 한국어 규칙 참고서입니다.

- `docs/wiki-crawl/*` (공식 위키 핵심 규칙 페이지)
- `docs/wiki-crawl-trouble-brewing/*` (Trouble Brewing + 캐릭터 상세 페이지)
- `docs/game-rules-reference.md`
- `docs/게임의 규칙.md`

기준 에디션: `Trouble Brewing` (기본판/입문 스크립트)

## 1. 게임 개요

- 진영: 선(Good) vs 악(Evil)
- 악의 핵심은 데몬(Demon), 선의 핵심 목표는 데몬 처형
- 게임은 밤(Night)과 낮(Day)을 반복
- 스토리텔러(진행자)가 규칙 판정과 정보 전달을 담당

## 2. 승리 조건 핵심

- 선 팀: 살아있는 데몬을 처형하면 즉시 승리
- 악 팀: 보통 생존 인원이 줄어 최종 국면에서 선이 데몬을 찾지 못하도록 만들면 승리
- 실제 종료 판정은 스토리텔러가 현재 상태(데몬 교체, 생존자 수, 특수 조건)를 함께 확인해 선언

## 3. 게임 진행 순서

1. 스크립트와 인원 확정
2. 역할 배정 및 시작 정보 전달
3. 첫날 전 밤 처리(첫밤 능력)
4. 낮 토론, 지목, 투표, 처형
5. 다음 밤 처리
6. 승리 조건 만족까지 반복

## 4. 낮 규칙 상세

- 토론: 공개 토론 + 필요 시 개인 대화
- 지목(Nomination): 처형 후보를 올리는 선언
- 투표(Vote): 스토리텔러가 순서대로 집계
- 처형(Execution): 득표 기준을 만족한 후보를 낮 종료 시 사망 처리
- 사망자도 발언은 가능하지만 투표 권한은 제한(유령표 1회 규칙 운용)

## 5. 밤 규칙 상세

- 스크립트 밤 순서를 반드시 지켜 처리
- 능력은 "누가 먼저 깨는지"에 따라 결과가 달라짐
- 밤 사망은 다음 날 아침 공개
- 정보형 캐릭터는 드렁큰/중독 등으로 오정보를 받을 수 있음

## 6. 상태(States) 핵심

- Drunk(취함): 본인 능력이 오작동, 본인은 취함 사실을 모를 수 있음
- Poisoned(중독): 중독 동안 능력 비정상 작동
- Dead(사망): 일반적으로 능력/투표 제약, 발언은 가능
- Alive(생존): 일반 규칙대로 지목/투표/능력 수행
- Madness(광기): 특정 스크립트에서 주장 강제 상태(기본판에서는 핵심 아님)

## 7. 능력 판정 원칙

1. 캐릭터 문구 원문 우선
2. 공식 위키의 해당 캐릭터/규칙 해설 반영
3. 충돌 시 스토리텔러 일관 판정 유지

실무적으로 중요한 점:

- "정보를 받는 능력"은 항상 진실이 아닐 수 있음
- 드렁큰/중독/특수 등록(registration) 효과를 항상 같이 의심
- 같은 상황에 판정을 바꾸지 않는 것이 공정성에 중요

## 8. Trouble Brewing 캐릭터 상세

아래 내용은 `docs/wiki-crawl-trouble-brewing/*.md` 기준으로 정리한 구현 친화 상세 규칙입니다.

공통 에러 방지 원칙:

- `능력 발동 시점`과 `정보 전달 시점`을 분리 기록한다.
- 조사 정보는 항상 `등록(registration)` 왜곡 가능성을 함께 저장한다.
- 죽음 처리 시 `죽인 원인(데몬 공격/처형/특수효과)`을 로그에 남긴다.
- "선택 가능한 대상"이 생존자만인지, 사망자 포함인지 캐릭터마다 강제 검증한다.

### 8.1 Townsfolk (13)

`Washerwoman`
- 능력 요약: 첫날 밤, 두 플레이어를 제시받고 그중 한 명이 특정 Townsfolk임을 안다.
- 스토리텔러 처리: 첫밤에만 정보 전달. 두 후보는 항상 서로 다른 플레이어여야 한다.
- 에러 방지: "정확히 한 명이 맞는지"가 아니라 "그중 한 명이 맞다" 포맷을 유지해야 한다.

`Librarian`
- 능력 요약: 첫날 밤, 두 플레이어를 제시받고 그중 한 명이 특정 Outsider임을 안다. 상황에 따라 Outsider 부재 정보를 줄 수 있다.
- 스토리텔러 처리: 바론으로 인해 Outsider 수가 바뀐 경우 반영해서 정보 구성.
- 에러 방지: 세팅 확정 전에 Librarian 정보를 만들면 안 된다. 반드시 최종 역할 배치 뒤에 산출.

`Investigator`
- 능력 요약: 첫날 밤, 두 플레이어를 제시받고 그중 한 명이 특정 Minion임을 안다.
- 스토리텔러 처리: Recluse/Spy 같은 등록 왜곡 가능성을 염두에 두고 정보를 줄 수 있다.
- 에러 방지: "미니언 실체"와 "미니언으로 등록됨"을 구분해 기록하지 않으면 재현 불가 버그가 난다.

`Chef`
- 능력 요약: 첫날 밤, 이웃한 악 플레이어 쌍의 수를 숫자로 안다.
- 스토리텔러 처리: 0 이상 정수로 전달. 연속 3명 악이면 쌍 2개로 계산.
- 에러 방지: 원형 좌석의 양끝 인접을 누락하지 말 것(마지막-첫번째도 인접).

`Empath`
- 능력 요약: 매일 밤, 양옆 이웃의 악 인원 수(0/1/2)를 안다.
- 스토리텔러 처리: 사망 여부와 무관하게 "이웃 좌석" 기준은 유지된다.
- 에러 방지: 밤마다 좌석이 아니라 생존 순서로 이웃을 재계산하면 오판정.

`Fortune Teller`
- 능력 요약: 매일 밤 2명을 선택해 둘 중 데몬 존재 여부를 안다. 선 1명은 Red Herring으로 데몬처럼 잡힌다.
- 스토리텔러 처리: Red Herring은 게임 시작 시 1회 고정 선택, 이후 변경하지 않는다.
- 에러 방지: Red Herring을 밤마다 재선정하는 실수가 가장 치명적이다. 고정 토큰을 별도 저장.
- 에러 방지: 선택 대상은 본인/사망자 포함 가능. UI에서 막지 말 것.

`Undertaker`
- 능력 요약: 매일 밤, 전날 처형된 플레이어의 실제 캐릭터를 안다.
- 스토리텔러 처리: "처형이 없었던 날"에는 정보 없음 처리.
- 에러 방지: 밤 사망자 캐릭터를 보여주면 안 된다. 반드시 "전날 처형자"만 조회.

`Monk`
- 능력 요약: 매일 밤 1명을 보호해 데몬 공격 사망을 막는다.
- 스토리텔러 처리: 보호 대상은 그 밤 데몬 킬 해석 직전에 확정.
- 에러 방지: 보호를 "모든 사망 면역"으로 처리하면 버그. 데몬 공격 방어로 한정.

`Ravenkeeper`
- 능력 요약: 밤에 죽으면 그 밤 1명을 선택해 캐릭터를 안다.
- 스토리텔러 처리: "밤 사망" 트리거인지 확인 후에만 발동.
- 에러 방지: 낮 처형 사망에도 발동시키는 실수를 방지해야 한다.

`Virgin`
- 능력 요약: 처음으로 자신을 지목한 사람이 Townsfolk면, 지목자가 즉시 처형될 수 있다.
- 스토리텔러 처리: 트리거는 게임당 1회성 이벤트로 잠금.
- 에러 방지: "처음 지목" 여부 상태 플래그가 없으면 중복 발동 버그 발생.

`Slayer`
- 능력 요약: 게임당 1회, 낮에 1명 지목. 대상이 데몬이면 즉시 사망.
- 스토리텔러 처리: 낮 공개 선언 시점에만 사용 가능.
- 에러 방지: 사용 횟수 플래그를 투표 종료 시점이 아닌 "선언 즉시" 소모해야 중복 클릭 방지.

`Soldier`
- 능력 요약: 데몬 공격으로는 죽지 않는다.
- 스토리텔러 처리: 다른 원인(처형, 특수효과) 사망은 가능.
- 에러 방지: 사망 원인 enum 없이 boolean 처리하면 Soldier 판정이 꼬인다.

`Mayor`
- 능력 요약: 최종 3인 국면 등 특정 조건에서 선 승리를 유도할 수 있다.
- 스토리텔러 처리: 데몬의 공격 대상 재지정(튕김) 판정을 일관되게 적용.
- 에러 방지: 엔드게임 판정 로직을 "데몬 생존 여부만"으로 두면 메이어 규칙 누락.

### 8.2 Outsiders (4)

`Butler`
- 능력 요약: 특정 플레이어에게 종속된 투표 제약이 생긴다.
- 스토리텔러 처리: 버틀러가 누구를 주인(master)으로 두는지 추적.
- 에러 방지: 투표 단계에서만 제약 검증해야 하며, 토론/지목까지 제한하면 오구현.

`Drunk`
- 능력 요약: 본인은 Townsfolk라고 믿지만 실제로는 Outsider이고 능력이 오작동한다.
- 스토리텔러 처리: 드렁큰의 "가짜 Townsfolk 정체"를 비공개로 관리.
- 에러 방지: 클라이언트에 실제 Drunk 타입이 노출되면 게임이 붕괴하므로 절대 비공개.

`Recluse`
- 능력 요약: 선이지만 악/데몬으로 등록될 수 있다.
- 스토리텔러 처리: 조사 결과에서 필요 시 악으로 취급 가능.
- 에러 방지: "실제 정렬"과 "등록 정렬" 필드를 분리하지 않으면 Investigator/Fortune Teller와 충돌.

`Saint`
- 능력 요약: 성인이 처형되면 선 팀 즉시 패배 조건이 발동될 수 있다.
- 스토리텔러 처리: 처형 결과 발표 전에 승리 판정 순서를 명확히 유지.
- 에러 방지: "사망 후 일괄 승리 체크"만 두면 성인 특수 패배 타이밍을 놓친다.

### 8.3 Minions (4)

`Poisoner`
- 능력 요약: 매일 밤 1명을 중독시킨다.
- 스토리텔러 처리: 중독 시작/종료 시점을 밤 단위로 명확히 기록.
- 에러 방지: 중독 지속기간을 무기한으로 두면 심각한 밸런스 버그가 된다.

`Spy`
- 능력 요약: 그리모어 정보를 볼 수 있고, 선으로 등록될 수 있다.
- 스토리텔러 처리: Spy의 등록 왜곡이 조사 결과에 반영되는지 일관 적용.
- 에러 방지: 관리자 UI에만 보여야 할 완전정보가 플레이어 UI로 전파되지 않도록 채널 분리.

`Scarlet Woman`
- 능력 요약: 보통 생존자 5명 이상에서 데몬이 죽으면 새 데몬으로 승계할 수 있다.
- 스토리텔러 처리: 데몬 사망 직후, 게임 종료 전에 승계 조건을 먼저 확인.
- 에러 방지: 승리 체크 순서가 "종료 판정 -> 승계 판정"이면 잘못된 즉시 종료 버그 발생.

`Baron`
- 능력 요약: Outsider 수를 늘려 선 팀 조합을 약화한다.
- 스토리텔러 처리: 세팅 단계에서 역할 분포 표를 먼저 바론 기준으로 변환.
- 에러 방지: 게임 시작 후 동적으로 Outsider 수를 바꾸면 안 된다.

### 8.4 Demon (1)

`Imp`
- 능력 요약: 매일 밤(첫밤 제외) 1명을 죽인다. 자신을 죽이면 살아있는 Minion 1명이 새 Imp가 된다.
- 스토리텔러 처리: self-kill 발생 시 같은 밤에 새 Imp가 즉시 추가 행동하지 않도록 처리.
- 에러 방지: "킬 대상 선택 가능 범위"에 사망자를 포함해야 한다(블러프/킬 싱크 구현에 필요).
- 에러 방지: 스타 패스 후 팀/정보/승리조건 주체를 새 Imp로 즉시 갱신해야 상태 불일치가 없다.

## 9. Trouble Brewing 구현 체크리스트 (앱/도구 기준)

- 캐릭터 배정: 인원표 기반 자동/수동 배정
- 밤 순서 엔진: 각 역할 타이밍/대상 선택/결과 전달
- 상태 엔진: 생존/사망/중독/취함/등록 왜곡 처리
- 낮 엔진: 지목/투표/처형 및 동률/문턱값 규칙
- 정보 전달: 개인 비밀 정보와 전체 공지 분리
- 로그: 스토리텔러용 비공개 판정 로그 유지
- 승리 판정: 데몬 사망, 스칼렛 우먼 승계, 최종 생존자 조건 반영
