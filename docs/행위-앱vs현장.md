# Blood on the Clocktower — 호스트 앱 / 참가자 앱 / 현장 분류표

기준 스크립트: **Trouble Brewing**
전제 환경: **오프라인 현장, 의자·핸드폰만 있음**

---

## 역할 재정의

| 역할 | 설명 |
|------|------|
| **호스트 앱** | 게임 운영 전담. 밤에 참가자를 깨워 화면을 직접 보여주는 방식으로 정보 전달 |
| **참가자 앱** | 수동 참고 도구. 본인 역할 카드·메모장·표기 도구. 게임 흐름에 능동 개입 없음 |
| **현장** | 말·손짓으로 처리. 앱 불필요 |

### 핵심 전제: 밤 시간 정보 전달 방식

> 참가자는 밤에 눈을 감고 있어 본인 핸드폰을 볼 수 없다.
> → 호스트가 직접 해당 참가자를 깨우고, **호스트 핸드폰 화면을 참가자에게 돌려서** 정보를 보여준다.
> → 참가자가 선택해야 할 경우도 **호스트 화면에서 참가자가 직접 탭**하거나, 호스트가 손가락으로 가리키며 확인하는 방식.

> 목적: 말로 오가는 정보를 빠르게 기록해 추리 근거로 활용

#### 이모지 시그널 기능 (낮 비언어 소통) — 옵션 A: P2P 전송

토론 중 구두로 말하기 어려울 때, 특정 참가자 핸드폰으로 이모지를 직접 전송.

| 시그널 예시 | 의미 |
|------------|------|
| 👍 | 동의 / 저도 같은 생각 |
| 🤫 | 나중에 따로 얘기하자 |
| 👀 | 저 사람 주목해 |
| ❓ | 방금 말 이해 못했어 |
| 🔪 | 저 사람 데몬 같아 |
| 🛡️ | 저 사람 선한 것 같아 |

**전송 방식:**
- 수신자: 특정 참가자 1명 지목 (본인 화면에만 표시, 다른 사람 안 보임)
- 또는 전체 브로드캐스트 (모두에게 표시)
- 이모지는 수신자 화면에 잠깐 팝업 후 사라짐

**P2P 구현 기반:**
- 이미 Trystero(Nostr) 기반 P2P 연결 구현 완료 (`src/js/p2p.js`)
- 방 코드 6자리로 호스트-참가자 전원 동일 room에 연결된 상태
- `p2p.js`에 `EMOJI_SIGNAL` 메시지 타입 추가로 구현 가능
- `sendToPeer(peerId, 'EMOJI_SIGNAL', { emoji })` 또는 `broadcast('EMOJI_SIGNAL', { emoji, from })` 활용

---

> **참가자 앱 설계 원칙:**
> - 밤에는 사용하지 않음. 낮 전용 도구.
> - 게임 상태(생존/사망 등)는 호스트 앱이 정본. 참가자 앱 기록은 어디까지나 개인 메모.
> - P2P 연결은 이모지 시그널 전용. 게임 판정 데이터는 호스트 앱만 처리.
