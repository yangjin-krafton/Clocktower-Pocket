# TTS 작성 규칙

이 폴더의 TTS CSV는 `밤에 랜덤 재생되는 역할 에피소드`를 만들기 위한 것이다.

기존처럼 한 편의 긴 연속 서사를 쓰는 방식이 아니라, 각 역할마다 독립적인 짧은 에피소드를 만든다.

## 1. 기본 방향

- 게임 본편의 밤에 틀어 줄 용도다.
- 플레이어가 눈을 감고 있는 동안, 분위기와 몰입을 유지해 주는 역할을 한다.
- 지루하지 않게, 하지만 게임 진행을 방해하지 않게 만든다.
- 각 에피소드는 서로 독립적이어야 한다.
- 배경 세계관 설명은 별도의 이야기로 분리한다.

## 2. 구성 원칙

- 역할 1개당 에피소드 1개를 기본으로 한다.
- 에피소드는 특정 역할의 능력, 심리, 긴장감, 선택의 무게를 보여줘야 한다.
- 에피소드는 한 인물의 짧은 소설처럼 들려야 한다.
- 글의 중심은 규칙 설명이 아니라, 이 세계 안에서 살아가는 인물의 밤 이야기여야 한다.
- 역할 능력은 직접 설명하지 말고, 행동과 시선과 사건으로 자연스럽게 연상되게 쓴다.
- 한 에피소드 안에서 시작, 긴장, 마무리가 있어야 한다.
- 다음 에피소드를 반드시 이어 받을 필요는 없다.
- 밤에 랜덤으로 재생해도 내용이 자연스럽게 들려야 한다.

## 3. 화자 규칙

- 화자 이름은 `docs/story-scenarios.md`에 있는 한국어 역할 이름을 사용한다.
- 역할명이 아닌 문장은 `나레이션` 또는 `전환`만 사용한다.
- 영어 역할명은 사용하지 않는다.
- 화자 표는 아래 이름만 사용한다.
- 하나의 에피소드에서는 `나레이션`을 포함해 최대 3명 화자만 사용한다.
- 다만 장면 연출상 꼭 필요할 때는 `손님`, `아이`, `노인`처럼 정체가 고정되지 않은 보조 인물을 제한적으로 사용할 수 있다.
- 보조 인물은 장면을 받쳐 주는 역할만 해야 하며, 중심 인물은 항상 해당 역할이어야 한다.
- 보조 인물의 대사는 1~2회, 짧은 확인 질문 정도로 제한한다.
- 보조 인물이 길게 설명하면 밤 배경 TTS가 아니라 대화극처럼 들리므로 피한다.

| 분류 | 한국어 역할 이름 |
|---|---|
| 정보 역할 | 세탁부, 사서, 조사관, 요리사, 공감인, 점쟁이, 장의사, 까마귀 사육사 |
| 보호 역할 | 수도사, 군인, 처녀, 처단자, 시장 |
| 오염 역할 | 독약꾼, 스파이, 진홍의 여인, 남작, 임프 |
| 흔들림 역할 | 집사, 주정뱅이, 은둔자, 성자 |
| 안내 화자 | 나레이션, 전환 |
| 보조 화자 | 손님, 아이, 노인 |

## 4. 작성 방식

- 한 줄에 한 화자를 유지한다.
- 같은 화자가 연속으로 말하는 부분은 줄바꿈하지 않고 한 줄에 이어 쓴다.
- 화자가 바뀌면 다음 CSV 행으로 분리한다.
- 짧은 대사와 짧은 설명을 섞어서 TTS 리듬을 만든다.
- 너무 긴 독백보다, 역할의 느낌이 살아나는 문장을 우선한다.
- 설명문처럼 쓰지 말고, 인물의 기억, 습관, 시선, 두려움, 결심이 들리게 쓴다.
- 듣는 사람이 규칙을 몰라도 장면이 떠오르게 써야 한다.

## 5. 밤 재생 기준

- 밤 행동 중에 자연스럽게 들릴 수 있어야 한다.
- 너무 복잡한 정보 설명은 피한다.
- 플레이어가 집중을 놓치지 않도록 문장을 간결하게 유지한다.
- 긴장, 기다림, 속임수, 보호, 확인 같은 감정을 중심으로 쓴다.
- 실제 게임의 단서나 정답처럼 들리지 않게 한다.
- 듣는 사람이 추리를 시작하지 않아도 되게, 분위기와 장면에 머물게 쓴다.

## 6. 이야기 분리

- 역할별 에피소드와 별개로, 세계관 소개용 독립 이야기 파일을 둘 수 있다.
- 세계관 이야기는 본편 밤 재생용 에피소드와 섞지 않는다.
- 세계관 이야기는 분위기, 배경, 마을의 공기만 담당한다.
- 역할별 에피소드는 게임 중 밤에 바로 틀 수 있어야 한다.

## 7. 권장 컬럼

- `script_id`
- `version`
- `audience_size`
- `episode`
- `order`
- `speaker`
- `text`

## 8. 예시 구조

```csv
script_id,version,audience_size,episode,order,speaker,text
tb-washerwoman-01,A,5-6,night,1,나레이션,밤이 시작됩니다.
tb-washerwoman-01,A,5-6,night,2,세탁부,누군가가 틀린 이름을 남겼습니다.
tb-washerwoman-01,A,5-6,night,3,전환,조금 더 기다리면 단서가 드러납니다.
```

## 9. 적용 기준

- 새 CSV를 만들 때는 역할별 단일 에피소드 구조를 우선한다.
- 기존 CSV를 수정할 때도 밤 재생용 독립성 기준을 맞춘다.
- 줄거리의 연결성보다, 한 번 들었을 때의 몰입감과 분위기를 우선한다.
